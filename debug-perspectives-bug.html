NewsPrism — Perspectives Bug Analysis

Systematic Debugging Report  ·  2026-05-20  ·  Branch: main

Summary

Every multi-source cluster on the live report (192.168.10.5:8080) shows the identical perspective text for every source:

视角差异：该来源报道与主摘要角度接近，未稳定提炼出可单列的差异化视角。 / 该来源报道与主摘要角度接近，未稳定提炼出可单列的差异化视角。

This is _fallback_perspective_text() — the string used when the LLM returns no perspective data for a source. The OPT-2 batch summarisation feature shipped with a prompt that tells the LLM "same rules as single-cluster mode" but never actually includes those rules, so the LLM always returns perspective_groups: [] and every source falls back.

Phase 2

Root Cause

There are two issues working together. The primary one is in the batch prompt; the secondary one is a stale style guide that compounds it.

Primary — batch prompt missing perspective_groups rules

Single-cluster prompt (summarizer.py:1313–1327, _build_prompt()) tells the LLM exactly what to do: 

- perspective_groups: 数组，包含去重后的 distinct perspective groups。
  每个元素为 {sources: ['来源A', '来源B'], perspective: '一句话共享视角'}
额外要求：
1. headline 和 body 只负责概括事件事实，不要在 body 里重复列举来源视角。
2. 所有来源必须且只能在 perspective_groups 中出现一次。
3. 当多个来源的视角实质相同，即使来自不同国家，也应合并到同一 group。
4. 只有明显不同的立场、强调点或国家视角，才拆成不同 group。

Batch prompt (summarizer.py:237–247, _batch_summarize()) has only one line about perspectives: 

- "每个集群的规则与单集群模式完全相同（多视角时填充 perspective_groups，单来源时 perspective_groups 为 []）。\n"

The phrase "same rules as single-cluster mode" is meaningless in context — the LLM does not know what those rules are. The format example in the prompt shows "perspective_groups": [...] but never shows what goes inside the array.

Secondary — style guide describes the old bullet format compounding

config/style-guide.md (loaded as the system prompt for every LLM call) still instructs: 

FORMAT FOR CLUSTER SUMMARY:
3. For each source with a distinct angle, write one bullet starting with
   the source name in【】brackets.

The single-cluster user prompt explicitly overrides this by saying "body 只负责概括事件事实，不要在 body 里重复列举来源视角". The batch prompt never overrides it. So the LLM correctly follows the system prompt — puts perspectives as bullet text in body — and returns perspective_groups: [] because it received no instruction to fill it.

Phase 3

Fix

Fix 1 — summarizer.py lines 237–247 primary fix

Replace the vague single-line in _batch_summarize() with the full perspective_groups rules, mirroring _build_prompt(): 

 prompt = (
     f"为以下 {len(clusters)} 个新闻事件集群分别生成摘要。\n\n"
     "输出 JSON 格式：\n"
-    "{\"clusters\": [{\"index\": 0, \"headline\": \"...\", \"body\": \"...\", "
-    "\"short_topic_name\": \"...\", \"topic_icon_key\": \"...\", \"perspective_groups\": [...]}, ...]}\n\n"
-    "每个集群的规则与单集群模式完全相同（多视角时填充 perspective_groups，单来源时 perspective_groups 为 []）。\n"
-    "index 字段必须与输入顺序一致（从 0 开始）。\n"
-    "只输出 JSON，不要解释。\n\n"
+    "{\"clusters\": [{\"index\": 0, \"headline\": \"...\", \"body\": \"...\", "
+    "\"short_topic_name\": \"...\", \"topic_icon_key\": \"...\", "
+    "\"perspective_groups\": [{\"sources\": [\"来源A\", \"来源B\"], \"perspective\": \"一句话共享视角\"}, ...]}, ...]}\n\n"
+    "每个集群的规则：\n"
+    "- headline: 一句话点明核心事件\n"
+    "- body: 2–4句客观事实总结，不要在 body 里列举来源视角\n"
+    "- short_topic_name: 4-10个中文字符的短专题名\n"
+    f"- topic_icon_key: 只能从以下列表中选择：{', '.join(self.topic_icon_allowlist)}\n"
+    "- perspective_groups: 去重后的视角分组数组。"
+    "每个元素为 {\"sources\": [\"来源A\"], \"perspective\": \"一句话视角\"}\n"
+    "额外要求：\n"
+    "1. 所有来源必须且只能在 perspective_groups 中出现一次。\n"
+    "2. 当多个来源视角实质相同，合并到同一 group；只有明显不同的立场才拆成不同 group。\n"
+    "3. 单一来源的集群 perspective_groups 为 []。\n"
+    "4. index 字段必须与输入顺序一致（从 0 开始）。\n"
+    "5. 只输出 JSON，不要解释。\n\n"
     "---\n\n"
     f"{clusters_text}"
 )

Fix 2 — config/style-guide.md lines 14–18 secondary fix

Replace the old bullet format instruction with a clarification that perspectives are output as structured perspective_groups in JSON — not bullets in body: 

- FORMAT FOR CLUSTER SUMMARY:
- 1. Open with a one-sentence topic headline (bold).
- 2. Write 2–4 sentences of factual summary covering the core event/development.
- 3. For each source with a distinct angle, write one bullet starting with the source
-    name in【】brackets.
-    Use the source's original name (e.g., 【Ars Technica】【Heise】【IT之家】【Tweakers】).
+ FORMAT FOR CLUSTER SUMMARY (JSON output fields):
+ - headline: one sentence naming the core event (bold)
+ - body: 2–4 sentences of factual summary only — do NOT include per-source
+   perspective bullets here; perspectives go exclusively in perspective_groups
+ - perspective_groups: structured array — each element is
+   {"sources": ["Source A", "Source B"], "perspective": "one-sentence shared angle"}
+   All sources must appear exactly once across all groups.
